  iDroid emergency badge — mark/unmark a mission code
    SetMissionEmergency(missionCode, enabled)

      Marks/unmarks a mission code as an emergency mission so it appears in
      the iDroid emergency-missions list. Accepting it runs the engine's
      vanilla emergency-mission flow: deploy picker is shown, prior mission
      is marked Aborted, "Return to Mission" works on game-over.

  Emergency popup text override (single-instance — not tied to a mission code)
    SetEmergencyMissionPopup       (title,      body)       -> bool   -- literal text
    SetEmergencyMissionPopupLangId (titleLabel, bodyLabel)  -> bool   -- LangId labels
    ClearEmergencyMissionPopupOverride()

      The "Emergency Missions" popup is a single generic notification the
      game shows once per iDroid session ("you have emergency missions; OK
      opens the list"). Use these to override its text:

        - SetEmergencyMissionPopup       — pass literal display strings
        - SetEmergencyMissionPopupLangId — pass LangIds (game looks up
          the localized text)

      Each call replaces any previous override (there's only one slot, not
      a queue). Clear restores the engine default.

      Pass nil for either field to leave just that field at engine default.
      Unknown LangIds silently fall back to engine default for that field.

      Examples:
        -- vanilla emergency mission, sortie prep off (default)
        V_TppUiCommand.SetMissionEmergency(10115, true)

        -- vanilla emergency mission with sortie prep on (engine deploy picker)
        V_TppUiCommand.SetMissionEmergency(10115, true, true)

        -- custom popup text (literal)
        V_TppUiCommand.SetEmergencyMissionPopup("Mission Alert", "Hostile transport spotted nearby.")

        -- custom popup text (LangId)
        V_TppUiCommand.SetEmergencyMissionPopupLangId(nil, "announce_heli_arrive_LZ")

        -- revert to engine-default popup text
        V_TppUiCommand.ClearEmergencyMissionPopupOverride()

  HUD mission icon — upper-left "urgent_time" widget
    ShowMissionIcon(title, body, time)

      Shows the urgent-mission icon in the upper-left HUD with custom
      title and body text.

      title  : LangId string, or nil for engine default
               ("hud_emergency_mission")
      body   : LangId string, or nil for default placeholder
      time   : seconds visible (default 6)

      The title and body text shown are whatever your LangId resolves to in
      the game's localization. Pass any LangId that the game knows — the
      same ones IH/Tpp use for other UI text work here. Made-up labels that
      aren't registered will not display anything (silent fallback).

      The title override persists between calls. To revert to the engine
      default title, pass nil:

        V_TppUiCommand.ShowMissionIcon(nil, "announce_refresh", 6)

      The icon's accent color is baked into the engine "urgent_time"
      preset and cannot be changed via this call.

      Examples:
        V_TppUiCommand.ShowMissionIcon("announce_heli_arrive_LZ", "announce_refresh", 6)
        V_TppUiCommand.ShowMissionIcon("hud_emergency_mission", "hud_fob_defense", 5)
        V_TppUiCommand.ShowMissionIcon(nil, nil, 4)

  HUD UI palette — runtime color overrides for mb_col_palette entries
    SetUiPaletteColor(key, r, g, b [, a])  -> bool
    RestoreUiPalette([key])

      Override a palette color used by HUD/menu widgets.

      key   : palette key string (e.g. "COL-3") or its 32-bit hash
      r,g,b : 0.0 .. 1.0
      a     : optional 0.0 .. 1.0 (default 1.0)

      Returns true if the color was applied, false if the palette key
      wasn't found.

      Important: the palette is only in memory once the iDroid (or the UI
      that owns the palette) has been opened at least once. Calling this
      from your mod's Init() is too early — call it the first time the
      player opens the relevant UI, or off a SendMessage hook for iDroid
      open.

      How it sticks: SetUiPaletteColor does two things in one call —
        (1) patches the matching color in every loaded UiPaletteData
            currently in memory (VirtualProtect + RGBA write), and
        (2) registers an in-process override so any FUTURE
            UiPaletteManager::GetForName lookup for that key returns your
            color regardless of what is in the palette data.
      Transient widgets (popups, notification icons, anything created on
      demand) pick up the new color on their next construction because of
      (2). Persistent widgets that were already built keep their cached
      value until they rebuild — opening and closing the pause menu is the
      cheapest way to force that rebuild.

      For mods shipping their own .fox2 palettes that might share key
      names (e.g. "COL-1") with other palettes, define a unique marker
      key in your .fox2 (e.g. "VFW_ANCHOR_MyMod_42") and set it first —
      subsequent SetUiPaletteColor calls will then target your palette.

      RestoreUiPalette() restores all originals you've overridden.
      RestoreUiPalette(key) restores just one key (string or hash). Both
      the in-memory palette write and the in-process override are cleared
      for that single key; everything else stays patched.

      Examples:
        -- typical (open iDroid first!)
        V_FrameWork.SetUiPaletteColor("COL-3", 1.0, 0.2, 0.2, 1.0)

        -- unique anchor pattern for modded palettes
        V_FrameWork.SetUiPaletteColor("VFW_ANCHOR_MyMod_42", 0, 0, 0, 0)
        V_FrameWork.SetUiPaletteColor("MyButton", 1, 0.5, 0)

        -- revert one key only
        V_FrameWork.RestoreUiPalette("cmn-col-emergency")

        -- revert everything
        V_FrameWork.RestoreUiPalette()


V_TppMission (custom mission positioning + emergency-state queries)
-------------------------------------------------------------------
    SetMissionStartPos(missionCode, x, y, z[, rotY])     -- optional, on-foot custom missions
    IsMissionEmergency(missionCode)                      -> bool

      To mark/unmark a mission as an emergency, use
      V_TppUiCommand.SetMissionEmergency under the V_TppUiCommand section
      above — it controls the iDroid badge / popup which is a UI concern.

      Examples:
        -- custom on-foot mission with explicit spawn position
        V_FrameWork.SetMissionStartPos(42000, -11.78, 8.48, 165.55, 0)
        V_TppUiCommand.SetMissionEmergency(42000, true)
